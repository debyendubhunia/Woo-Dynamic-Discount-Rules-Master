TECHNICAL ANALYSIS & USAGE REPORT: AI SUBSYSTEM
================================================
Date: June 6, 2026
Author: Lead Integration Architect

1. ARCHITECTURAL OVERVIEW
This document outlines the design and runtime execution details of the AI Recommendation Center added to the Woo Dynamic Discount Rules Master plugin. The core objective of this subsystem is to identify optimization opportunities across the storefront's catalog without introducing heavy database locks or external runtime dependencies during the checkout flow.

2. HEURISTIC & APRIORI IMPLEMENTATION DETAILS
Rather than relying on expensive real-time API queries for every customer interaction, the engine employs a dual-strategy design:

- Local Transaction Scanning (Apriori logic):
  The local engine executes a self-join query on the `woocommerce_order_items` and `woocommerce_order_itemmeta` tables to look for product co-occurrences. It filters pairs that have appeared together in at least 2 distinct orders (configurable threshold) and calculates co-occurrence frequencies to suggest product bundle rules (e.g., "Product A + Product B" with a 10% bundle discount).
  
- Inventory Velocity Models:
  The engine queries active product posts against the order tables to locate items with zero sales in the past 60 days (slow-moving) or items whose sales volume dropped by >30% month-over-month (declining sales). 

3. PROMPT DESIGN & EXTERNAL INTEGRATION
For merchants who enable external providers (OpenAI or Anthropic Claude), the plugin translates transaction trends into JSON-formatted analysis payloads. The prompt specifies structured outputs to ensure the response maps cleanly into the discount rule schema (name, type, discount value, and product filter lists).

4. PERFORMANCE & RESOURCE OPTIMIZATION
- Transient Caching: All calculations (both local queries and external API responses) are cached using WordPress Transients with a 6-hour TTL (`wddrm_ai_suggestions`). This ensures that page loads in the admin dashboard remain under 100ms.
- Non-blocking AJAX: The recommendations are fetched asynchronously via the `wddrm_get_ai_suggestions` AJAX action.

5. DISCOUNT ENGINE & ANALYTICS PIPELINE (June 2026 Update)
- Recalculation Resiliency:
  WooCommerce executes the `woocommerce_before_calculate_totals` hook multiple times within the same page load. To prevent compounding/double-discounting and price dilution (e.g. BOGO free items recalculating to 0 value), the engine implements a runtime cache (`wddrm_original_price`) on the cart item array. This resets product prices back to their database values at the start of each calculation cycle, ensuring correct discount amount tracking.
- Session-Based Order Tracking:
  Applied discount rule details (including rule ID, discount amount, and original subtotal) are tracked inside the calculation loop and stored in the WooCommerce session under `wddrm_applied_rules`. Upon checkout completion (`woocommerce_checkout_order_created`), these rules are parsed and persisted to the `{prefix}wddrm_rule_applications` database table, enabling accurate attribution.
- Currency Normalization:
  To avoid raw entity strings (e.g., `&#036;`) displaying in AJAX-driven dashboard UI nodes or raw CSV exports, the currency symbol is decoded to raw UTF-8 using `html_entity_decode(..., ENT_QUOTES, 'UTF-8')` before script localization and CSV formatting.
